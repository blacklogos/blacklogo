0.2 Start & Clone

Module 0: Getting Started | 15 min

Starting Claude Code

Now that you have Claude Code installed, it's time to start using it! In this lesson, you'll learn how to launch Claude Code and clone this course to your computer.

What You'll Do

  • Start your first Claude Code session
  • Understand the Claude Code interface
  • Clone the course materials
  • Explore the MarketFlow project

Launching Claude Code

Open your terminal or command prompt and navigate to where you want to work. Then run:

claude-code

You'll see a welcome message and the Claude Code prompt. This is where you'll interact with Claude!

You're In!

When you see the Claude Code prompt, you're ready to start. It should look something like:

Claude Code v1.0.0
Ready to help with your marketing tasks!

>

Understanding the Interface

Claude Code uses a conversational interface. You simply type what you want Claude to do, and it responds. Think of it as chatting with a very capable marketing assistant.

Basic Commands

Here are some essential commands to know:

  • /help - Get help and see available commands
  • /clear - Clear the conversation history
  • /exit or Ctrl+C - Exit Claude Code

Your First Interaction

Let's try your first command. In Claude Code, type:

Hello! I'm ready to learn Claude Code for marketing.

Claude will respond with a friendly greeting. This confirms everything is working correctly!

Setting Up Your Workspace

Create a dedicated folder for this course. In your terminal (outside of Claude Code), run:

mkdir claude-marketing-course
cd claude-marketing-course

Now start Claude Code from this directory:

claude-code

Why This Matters

Claude Code works within the directory where you launch it. By creating a dedicated folder, you keep all your marketing projects organized.

Creating the MarketFlow Project

Throughout this course, you'll work with MarketFlow, a sample marketing project. Let's set it up.

In Claude Code, type:

Create a folder structure for a marketing project called MarketFlow with folders for campaigns, content, research, and analytics.

Claude will create the folder structure for you. You can verify it worked by asking:

Show me the directory structure

Exploring Your Project

Now let's explore what we created. Ask Claude:

List all the folders in this project

You should see:

  • campaigns/ - For campaign briefs and plans
  • content/ - For marketing copy and content
  • research/ - For competitive analysis and market research
  • analytics/ - For data analysis and reports

Exercise: Create Your First File

Let's practice by creating a file. In Claude Code, type:

Create a file called README.md in the project root with a brief description of MarketFlow

Claude will create the file and write content for you. Then ask it to show you the file:

Show me the contents of README.md

Understanding How Claude Works

When you ask Claude to do something, it:

  1. Understands your request in natural language
  2. Plans the steps needed to complete the task
  3. Executes the necessary file operations
  4. Confirms what it did

You don't need to know any coding - just describe what you want in plain English!

Working with Files

Claude Code can read, write, edit, and organize files. Here are some things you can ask:

  • "Read the campaign brief file"
  • "Create a new content calendar"
  • "Edit this email copy to be more engaging"
  • "Move this file to the campaigns folder"
  • "Find all files mentioning 'product launch'"

Best Practices

As you work with Claude Code, keep these tips in mind:

  • Be specific: "Create a Facebook ad" is vague. "Create a Facebook ad for our new product targeting young professionals" is better.
  • Provide context: The more context you give, the better Claude's output will be.
  • Iterate: If the first output isn't perfect, ask Claude to refine it.
  • Ask questions: If you're unsure about something, just ask Claude!

Remember

Claude Code works with files in your current directory. Always make sure you're in the right folder before starting a Claude Code session.

Saving Your Work

Everything Claude creates is saved to your local files automatically. You can:

  • View files in VS Code or any text editor
  • Share files with your team
  • Use version control (like Git) to track changes
  • Back up files to cloud storage
